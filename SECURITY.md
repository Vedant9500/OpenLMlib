# Security Policy

## Supported Versions
Security updates are provided for the latest release on the default branch.

## Reporting a Vulnerability
Please do not report security vulnerabilities in public issues.

Instead, report privately to the maintainers with:
- A clear description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested mitigation (if available)

## Response Process
- We will acknowledge receipt as soon as possible.
- We will validate and triage the report.
- We will prepare a fix and coordinate disclosure timing.
- We will publish a security note after remediation.

## Scope
This policy covers the repository code, release artifacts, and default deployment paths documented in this project.
